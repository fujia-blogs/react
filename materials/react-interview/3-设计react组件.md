## 基础

### 组件分类

1. 经典的分类：

- 把只作展示，独立运行，不额外增加功能的组件，称为哑组件，无状态组件或展示组件。
- 把处理业务逻辑与数据状态的组件称为有状态组件，灵巧组件。灵巧组件一定包含至少一个灵巧组件或展示组件。

**展示组件**的复用性更强，**灵巧组件**更专注于业务本身。

#### 组件设计模式

1. 展示组件

2. 业务组件

### 工程实践

> 通过目录级别完成切分

1. 将最基本的展示组件放入 basic 目录中。

**建议：basic 组件，用类似 storybook 的工具进行组件管理**

2. 将容器组件放入 container

3. 高阶组件放入 hoc 中

4. 将页面组件外层组件放在页面目录中
